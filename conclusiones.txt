CONCLUSIONES - Prueba Técnica Sofka con Karate

1. FUNCIONALIDAD DE LA PETSTORE

- La API pública de Swagger PetStore permite realizar operaciones CRUD sobre mascotas.
- Durante las pruebas se observó que el endpoint no siempre garantiza persistencia inmediata.
- Consultas por estatus (`/pet/findByStatus`) pueden no devolver los registros recién actualizados, lo que genera falsos negativos.

2. PROBLEMAS ENCONTRADOS

- **Sincronización**: tras crear o actualizar una mascota, algunas veces el GET por ID o por estatus devuelve 404 o no incluye el registro esperado.
- **Latencia**: la API presenta retrasos intermitentes en la propagación de datos.

3. MEJORAS IMPLEMENTADAS

- Se implementó **retry until** para manejar la latencia y consistencia eventual.
- Se separaron **features de utilidades** (ejemplo: `utils/retry.feature`) para mayor organización.
- Se incluyeron **tests data-driven** para probar múltiples combinaciones de entradas.
- Se configuraron **environments múltiples** (`dev`, `mock`) para soportar tanto la PetStore real como posibles ambientes simulados.
- Se etiquetaron los escenarios para facilitar la ejecución selectiva y manejo de casos inestables.


4. CONCLUSIÓN GENERAL

Karate resultó ser una herramienta adecuada para la automatización de APIs REST, con sintaxis simple y soporte integrado para validaciones, JSON, reintentos y reportes.  
El mayor reto fue la inestabilidad de la API PetStore pública, lo cual se mitigó con reintentos y buenas prácticas.
